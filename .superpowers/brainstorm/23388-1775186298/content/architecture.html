Animation System Architecture

How the new components fit into the existing render loop

// main loop — each frame
dt = clock.restart();
snapshot = engine.getStateSnapshot();

renderer.detectDelta(snapshot, animMgr); // diff → enqueue animations + sounds
animMgr.update(dt); // advance all active animations
soundMgr.flush(); // play any pending sound events

renderer.render(snapshot, humanId, animMgr); // static world, skipping masked elements
animMgr.draw(window); // in-flight elements on top
inputHandler.drawButtons(snapshot);
window.display();
GameRenderer (modified)
• Stores m_prevSnapshot
• detectDelta() diffs snapshots → enqueues into AnimationManager
• render() queries animMgr.isMasked() to skip elements still in flight
• No animation state of its own
AnimationManager (new)
• Owns std::vector<Anim> queue
• update(dt) advances progress 0→1
• draw(window) renders sprites/text at interpolated positions
• isMasked(key) — tells renderer what not to draw statically
• enqueue(Anim) with optional delay offset for sequencing
SoundManager (new)
• Loads WAV/OGG buffers at startup
• post(SoundEvent) called from detectDelta
• flush() plays pending events (rate-limited to avoid overlap)
• Sounds: deal, chip, fold, win
