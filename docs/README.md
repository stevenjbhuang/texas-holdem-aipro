# Documentation

## Structure

```
docs/
├── spec/
│   └── design.md           # Full project design spec (architecture, components, data flow)

│   └── implementation.md   # Step-by-step implementation plan (added when ready)
```

## Reading Order

1. Start with `spec/design.md` — understand what we're building and why
2. Follow `plan/implementation.md` — guided tasks phase by phase
3. Refer to `guides/` — concept explanations for each topic as you encounter them
